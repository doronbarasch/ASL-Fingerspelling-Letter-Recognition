ASL Fingerspelling Letter Recognition

Collaborators: Doron Barasch, RanDair Porter

Towards the beginning of this project we found that there are, shockingly, essentially no commercially available copmuter-based ASL finger spelling interpreters. Living in a time where accessibility is paramount, we found this upsetting and wanted to make a step towards finding a possible solution. For this project we decided to take on the task of creating a program that would be able to interpret/translate standard ASL finger spelling letters in real time. Such a program requires using the images from video footage and classifying those as letters, which meant that we had to train a model on images of finger spelling letters in ASL.

We initially used a single dataset (the first dataset) to train the model and attempt to classify finger spelling letters in real time. However, the video finger spelling classification proved to be rather difficult, requiring us to combine three large datasets to create a model that would generalize well and accept, hopefully, as many correct finger letters as possible. As such, we combined all of the images, bar a test set from the second dataset and some modifications, from three datasets. We removed the C and O images from the first dataset because they were head-on images of the letters where the palm was facing outward rather than the shape of the letter facing outward.

These datasets can be found at the following links:

ASL Fingerspelling Images (RGB & Depth)
ASL Alphabet
Sign Language Gesture Images Dataset

To train our model, we made use of an altered version of the convolutional neural network that Professor Redmon made. We additionally used PyTorch to expedite dataset loading, splitting, and training as well as model saving. Using PyTorch, we applied transforms to images in the dataset to ensure that the model would generalize well. This transforms consisted of resizing the dataset images, making the images grayscale, randomly rotating the images slights, randomly changing the perspective of the images, and randomly flipping horizontally. We randomly flipped the images horizontally in order to account for both left- and right-handed signers.

We wrote the majority of the code ourselves, referencing Professor Redmon's PyTorch tutorial. The Python files with the code can be found at the project repository for this GitHub page.

We found that the best neural net, neural net 7, was the one that did all of the aforementioned transforms and used the combined dataset. The other neural nets used less or slightly different transforms, or less data, and resultantly did not perform as well.
